## 自我迭代协议 v2.0

> **核心理念**：
> - "**不做事后诸葛，要做未卜先知**"
> - 从"解释型迭代"升级为"**预测-验证闭环型迭代**"
> - 从"单案例反应"升级为"**规律集群演化**"
> - 从"主观星级"升级为"**置信度校准**"

### 9大机制

1. **预测密封协议** — 用户不说事实时，先封存预测再验证
2. **规律冲突检测器** — 新规律3道审查（一致性/覆盖度/可证伪）
3. **案例质量分级** — A/B/C/D四级，加权分≥2.4才晋级
4. **置信度校准系统** — 星级重定义为统计命中率
5. **八字指纹索引** — 相似命盘自动匹配历史案例
6. **置信度系数** — 信息缺失时量化警告
7. **敏感度分析** — 关键时辰边界前后对比
8. **自证计算链** — 排盘自校验，防幻觉
9. **神煞白名单** — 仅限29个已知神煞，禁止编造

### 规律晋级流程

```
新规律入库 → 3道审查 → 过审则标"🟡 待验证 X.X/2.4"
         ↓
     更多案例验证 → 加权累分
         ↓
     ≥2.4  → 🟢 晋级为已验证规律
     遇反例 → 🔴 进入冲突待判池
     持续证伪 → ⚫ 降为存疑/移除
```

## 如何使用

### 场景A：用户进行命理咨询

```
在支持 .kiro/skills 的 AI 工具中（如 Kiro IDE）：
1. 加载本仓库
2. 提供八字信息（年月日时+性别+城市）
3. 描述您的需求（综合/婚姻/合婚/择日）
4. AI自动进入对应模式，输出结构化分析
```

### 场景B：持续迭代skill

```
1. 提供新的八字案例 + 实战反馈
2. AI自动：
   - 回溯验证
   - 提取规律
   - 过3道审查
   - 更新规律分数板
   - 递增版本号
3. 可通过"密封预测"模式真实校准AI能力
```
