## Skill 支持的 7 种工作模式（v6.1.0 与主入口 §1.1.1 完全对齐）

| 模式 | 场景 | 调用模块 | 执行方法 |
|---|---|---|---|
| **M1 · 综合批命** | "分析我的八字" | A + B + C | 三步交互（Step 1 速览 → 反馈 → Step 3 深度） |
| **M2 · 婚姻专题** | "看看我的婚姻" | B + C（必先走 §1.4 婚况采集） | 三步交互（婚况前置） |
| **M3 · 合婚配对** | "帮我们合婚" | **D**（双盘交互层）+ A 双方 + B 双方 | 10 维矩阵 + 三垣呼应 + 桃花归宫识别 + 三窗口推荐 |
| **M4 · 剖腹产择日** | "给孩子选日子" | **E**（独立流程层） | 七层递进 + 三体系投票（盲派 1 + 子平 1 + 择吉 0.5）+ 12 段报告 |
| **M5 · 知识查询** | "什么是阴阳差错日" / "丙火像什么" / "我适合什么职业" | 附录-取象 / 附录-职业（按子主题） | Step 3 直接答 + 附录引用，无需 Step 1 排盘 |
| **M6 · 回溯验证** | "我去年离婚了，看看为什么" | A + B + C + 案例库回写迭代 | 大运流年三层互动 + 触发 calibration-log 写入 |
| **M7 · 密封预测** ⭐ | "先别告诉你事实，你先预测" | 占位（v6.2.0 实现） | 路由占位 → 入口 ack 到 `cases/predictions-sealed.md` |

> 模式判定优先级：M4 > M3 > M2 > M7 > M6 > M5 > M1（更专精的模式优先匹配）。
> 详见 `.kiro/skills/blind-bazi-analyst.md` §1.1.1 工作模式速查表与 §1.2 模块路由表。

## 自我迭代协议 v3.0（v6.1.0 升级 · 在 v2.0 协议基础上 +1 大机制）

> **核心理念**：
> - "**不做事后诸葛，要做未卜先知**"
> - 从"解释型迭代"升级为"**预测-验证闭环型迭代**"
> - 从"单案例反应"升级为"**规律集群演化**"
> - 从"主观星级"升级为"**置信度校准**"
> - **v3.0 新增**：从"被动回写"升级为"**主动入库 + 覆盖度自检**"

### 10 大机制（前 9 大保留自 v2.0 协议 · 第 10 大为 v6.1.0 新增）

1. **预测密封协议** — 用户不说事实时，先封存预测再验证
2. **规律冲突检测器** — 新规律 3 道审查（一致性 / 覆盖度 / 可证伪）
3. **案例质量分级** — A/B/C/D 四级，加权分 ≥ 2.4 才晋级
4. **置信度校准系统** — 星级重定义为统计命中率
5. **八字指纹索引** — 相似命盘自动匹配历史案例
6. **置信度系数** — 信息缺失时量化警告
7. **敏感度分析** — 关键时辰边界前后对比
8. **自证计算链** — 排盘自校验，防幻觉
9. **神煞白名单** — 仅限 29 个已知神煞，禁止编造
10. **md 文档自动入库协议**（**v6.1.0 新增** · 来源：`META/ingestion-protocol.md`）— 启动自检 → 五阶段协议 → 覆盖度矩阵更新 → 来源追溯三元组登记 → 闭环对接 cases-index / rule-conflicts / calibration-log

### 规律晋级流程

```
新规律入库 → 3 道审查 → 过审则标"🟡 待验证 X.X/2.4"
         ↓
     更多案例验证 → 加权累分（A=1.0 / B=0.6 / C=0.3 / D=0.1）
         ↓
     ≥ 2.4 → 🟢 晋级为已验证规律
     遇反例 → 🔴 进入冲突待判池
     持续证伪 → ⚫ 降为存疑/移除
```

### v3.0 新增机制（第 10 大）的入口与产物

| 角色 | 文件 | 职责 |
|---|---|---|
| 启动自检 | `blind-bazi-analyst.md` §0 | 每次 Skill 加载时识别新 / 修改 md，告警但不阻塞问命 |
| 五阶段协议 | `META/ingestion-protocol.md` §I.1 | 识别 → 抽取候选 → 三道审查 → 入库 → 晋级 |
| 覆盖度矩阵 | `META/module-coverage-matrix.md` | 25 行 × 16 列：每份根目录 md / 案例 md → 各模块吸收度 |
| 待办队列 | `META/pending-extraction-queue.md` | P0 / P1 / P2 候选规律（含 9 份高德臣 PDF P0 全量待办） |
| 来源追溯 | `META/source-trace.md` | 每条规律的『来源 md + 段落锚点 → 当前模块:章节号』三元组 |

## 如何使用

### 场景 A：用户进行命理咨询

```
在支持 .kiro/skills 的 AI 工具中（如 Kiro IDE）：
1. 加载本仓库
2. 提供八字信息（年月日时 + 性别 + 城市）
3. 描述您的需求（综合 / 婚姻 / 合婚 / 择日 / 知识查询 / 回溯验证 / 密封预测）
4. AI 自动进入对应模式（M1-M7），输出结构化分析
5. 涉及婚姻 / 子嗣 / 官运 / 寿限的 ★★★★★ 铁断必带"【前提：命主当前××状态】"标注
```

### 场景 B：持续迭代 Skill

```
1. 提供新的八字案例 + 实战反馈
2. AI 自动：
   - 启动自检（§0）扫描新增 / 修改的 md 文档
   - 触发 ingestion-protocol 五阶段协议
   - 回溯验证 + 提取规律 + 过 3 道审查
   - 更新规律分数板 + source-trace 三元组 + module-coverage-matrix
   - 递增版本号 + CHANGELOG 同步
3. 可通过 M7 密封预测模式真实校准 AI 能力（v6.2.0 实现）
```

### 场景 C：知识查询（v6.1.0 新增）

```
直接问"什么是 XX""丙火像什么""我适合什么职业"等知识类问题：
- 取象类 → 加载 appendix-quxiang.md
- 职业类 → 加载 appendix-zhiye.md
- 不必先排盘
```
