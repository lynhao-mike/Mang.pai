# 规律冲突待判池（Rule Conflict Pool · v6.0 同步）

> **v6.0 同步说明**：主 skill 模块化后，跨模块冲突单列追踪。
> **新增分类**：
> - **模块内冲突**：同一模块（A/B/C）内的两条规律相矛盾（按既有流程处理）。
> - **跨模块冲突**：例如"模块 B 给的格局等级"与"模块 C 给的应期窗口"互相打架。
>   → 跨模块冲突需要**主入口在 Step 3 输出前显式仲裁**，仲裁规则：
>     1. 模块 B（格局）定的"天花板"为硬约束 —— 模块 C 给的概率不得超过模块 B 等级允许的上限
>     2. 模块 A（基础作用）若发现"穿坏用神之根" —— 模块 B 与 C 的乐观结论必须**同步下调一级**
>     3. 用户反馈中的"事实修正"优先级 > 任何模块的内部规律

> 新规律与既有规律矛盾时，进入此池等待更多案例仲裁。
> 禁止在此池的规律直接用于断命，只能作为"可能性提示"。

---

## 处理原则

```
冲突规律 → 标"争议中" → 等第2个案例
       ├── 新案支持新规律 → 旧规律降级
       ├── 新案支持旧规律 → 新规律移除
       └── 都有支持 → 拆分适用场景
```

### 争议 #2：十恶大败日

| | 传统规律 | 新规律（#CASE-20260513-01） |
|---|---|---|
| **内容** | 一生贫穷，财库有漏 | 少年无祖业，自立更生；中晚年金水运可富 |
| **来源** | 传统命理古籍 | 1个A级案例（基于大运分析的推论，实际财富结果尚未验证） |
| **支持案例数** | （未建库前无量化） | 1 (A=1.0，但此案命主当前33岁，晚年富否尚未兑现） |
| **争议点** | 新规律的"验证"是**未来预测**而非**已发生事实**——不能算真验证 |

| **当前处理** | 新规律**降级为"假说"**，不计入晋级分数。必须有"43+岁已富"的实证案例才能晋级 |

## 仲裁触发

当某争议的任一方获得**累计加权分数≥1.8**（3个A级或等价）：
- 该方"胜出" → 另一方降级为"存疑"
- 胜出方从本池移出，进入主规律库

## 跨模块冲突（v6.0 新增分类）

### 跨模块冲突 #1：应期层级锚定偏差（模块C vs 模块B）

| | 模块 B 判定 | 模块 C 判定 |
|---|---|---|
| **内容** | 伤官合杀(杀轻伤重) + 丙子运印护 → 副科在丙子运内"轻松兑现" | 2026 丙午年 10 项铁证 → 88/100 标为"提拔副科年" |
| **冲突点** | B 说"轻松"→ 暗示不需要等到丙子运末期（2026）才到；C 把 2026 锚定为副科年 |
| **实际结果** | 命主 2025 前已是实职副科（改革督查专员），2026 时副科早已兑现 |
| **根因** | 模块 C 在做应期推演时，**未先与模块 B 的"轻松/困难"程度判定做交叉验证**。B 说"轻松"= 运初即可兑现，C 不应把应期推到运末 |
| **修正方案** | 模块 C 新增规则：*"当模块 B 标注'轻松兑现'时，应期窗口优先锚定在大运前 1/3 段而非中后段"* |
| **当前处理** | 2026 年 10 项铁证重新定性为"**正科启动年 / 更高层级**"而非副科年 |
| **触发案例** | #CASE-001（A级）第一轮反馈 |

### 跨模块冲突 #2：印运应期默认归类偏差（v6.0.2 新增）

| | 模型默认 | 实际兑现 |
|---|---|---|
| **内容** | 大运透印 / 寅午合印 → 默认归类为**学业 / 文件 / 任命**类应期 | #CASE-001 戊寅运（16-18岁）实际为"走体育生路线、成绩反差" |
| **冲突点** | 模型把"寅藏丙印"等同于"印运" → 但**戊寅大运中"戊"为透干劫财、"寅"中印仅藏**——劫财强于印 |
| **根因** | 模块 C 应期归类规则**只看大运地支藏的十神是否包含印**，未做"透干 vs 藏支"的力量优先级判断 |
| **修正方案** | 模块 C 新增规则（已写入 R-001-ADD-06）：**"当原局印星不透 + 大运透出的天干为劫财/比肩/伤官时，应期优先归类为劫财类（兴趣分流/合作/比劫夺财）而非印类（学业/任命）"** |
| **当前处理** | 在模块 C 的"应期事件类型清单"中**新增第 11 类「兴趣分流 / 跨界 / 副业」**，触发条件：劫财大运 + 印不透 + 伤官旺 |
| **触发案例** | #CASE-001（A级）第二轮反馈 |
