# 规律冲突待判池（Rule Conflict Pool · v6.0 同步）

> **v6.0 同步说明**：主 skill 模块化后，跨模块冲突单列追踪。
> **新增分类**：
> - **模块内冲突**：同一模块（A/B/C）内的两条规律相矛盾（按既有流程处理）。
> - **跨模块冲突**：例如"模块 B 给的格局等级"与"模块 C 给的应期窗口"互相打架。
>   → 跨模块冲突需要**主入口在 Step 3 输出前显式仲裁**，仲裁规则：
>     1. 模块 B（格局）定的"天花板"为硬约束 —— 模块 C 给的概率不得超过模块 B 等级允许的上限
>     2. 模块 A（基础作用）若发现"穿坏用神之根" —— 模块 B 与 C 的乐观结论必须**同步下调一级**
>     3. 用户反馈中的"事实修正"优先级 > 任何模块的内部规律

> 新规律与既有规律矛盾时，进入此池等待更多案例仲裁。
> 禁止在此池的规律直接用于断命，只能作为"可能性提示"。

---

## 处理原则

```
冲突规律 → 标"争议中" → 等第2个案例
       ├── 新案支持新规律 → 旧规律降级
       ├── 新案支持旧规律 → 新规律移除
       └── 都有支持 → 拆分适用场景
```

- 该方"胜出" → 另一方降级为"存疑"
- 胜出方从本池移出，进入主规律库

### 跨模块冲突 #1：应期层级锚定偏差（模块C vs 模块B）

| | 模块 B 判定 | 模块 C 判定 |

| **内容** | 伤官合杀(杀轻伤重) + 丙子运印护 → 副科在丙子运内"轻松兑现" | 2026 丙午年 10 项铁证 → 88/100 标为"提拔副科年" |
| **冲突点** | B 说"轻松"→ 暗示不需要等到丙子运末期（2026）才到；C 把 2026 锚定为副科年 |

## 系统性盲区登记（v6.0.3 新增分类）

> **定义**：不属于"两条规律打架"型冲突，而是**整层算法或自检环节缺失**——模型在该层从未被监督，错了也不会自报。
> **处理优先级**：高于普通规律冲突。盲区一旦确认，必须立刻补强算法 + 加入自检 + 回归历史案例。

### 盲区 #1：起运 / 交运算法层从未被校验过（v6.0.3）

| 项 | 内容 |
|---|---|
| **触发案例** | #CASE-001（5岁1月被取整为6岁，失真 ≈ 11 月）+ #CASE-002（"阴女顺排"描述错·结果偶合对） |
| **影响范围** | **所有大运 / 流年应期判断**——起运是地基，错 1 岁则后续每个流年的"所处大运"都可能错位 |
| **根因 1** | `module-a-paipan.md` 只写"阳男阴女顺排"原则，**未写"阴阳判定看年干，不看年支"**；导致"壬申""庚子"等阳干阴支年份易被误判为阴年 |
