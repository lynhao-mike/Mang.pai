## §P.0 队列规则

| 规则 | 说明 |
|---|---|
| **入队**（push） | 由 ingestion-protocol §I.1.4 阶段 4 自动触发（识别 + 分类 + 提取 → 写入本队列） |
| **出队**（pop） | 由 §P.3 三种条件之一触发：(a) 已晋级写入主模块 / (b) 累计加权 ≥ 2.4 / (c) 被证伪进 rule-conflicts.md 或降级存疑 |
| **顺序** | 按入队日期倒序追加（新条目在最上方）；P0 / P1 / P2 内部分桶 |
| **不可跳级** | 候选条目从 P2 升 P1 / P1 升 P0 必须有人审或新案例触发 |
| **合并** | 同一规律来自两个不同来源 → **合并条目**而非新建（在『来源文件』栏列出全部来源、累加分数） |

> **铁律 §P.0-A**：每条候选必带『来源段落锚点』栏（文件路径 + 段落起始关键词）。
> **铁律 §P.0-B**：候选等级用『0.0–2.4 数字』而非『黄灯 / 绿灯』，方便加权累计；状态用 🔴 阻塞 / 🟡 待验证 / ⚪ 待 OCR 三种符号。

## §P.1 队列主表（结构定义）

每条候选规律必须填齐以下 8 栏：

```
| id | 来源文件 | 来源段落锚点 | 候选规律内容 | 拟入模块 | 候选等级 | 等待条件 | 入队日期 |
```

字段说明：

| 栏 | 说明 |
|---|---|
| **id** | 风格 P0-NN / P1-NN / P2-NN（NN 入队序号） |
| **来源文件** | 完整路径（如 `案例库/案例002_错误修正与子女八字呼应铁证_2026-05-16.md`） |
| **来源段落锚点** | 章节号 + 段落起始关键词（如 `#论断三 + "戌合卯空亡 = 印章引动"`） |
| **候选规律内容** | ≤ 60 字摘要 |
| **拟入模块** | A / B / C / D / E / 取 / 职 / cases-XX 之一或多个 |
| **候选等级** | 0.0–2.4 数字（A 级单案 = 1.0；B 级 = 0.6；C 级 = 0.3；D 级 = 0.1；理论已引用 = 0.6；理论未引用 = 0.3；PDF 待 OCR = 0.0） |
| **等待条件** | 出队前置条件（见 §P.3） |
| **入队日期** | YYYY-MM-DD |

## §P.3 出队规则（三种条件，满足任一即出队）

### §P.3.1 (a) 已写入对应模块文件且 grep 可证

```
触发：阶段 5 完成后，模型在 .kiro/skills/modules/ 或 cases/ 中能 grep 到该候选规律的等价表述
出队动作：
  1. 该条目从本队列移除（或在 §P.4 历史记录区标『已出队 · 已写入主模块』）
  2. source-trace.md 追加三元组（来源 md / 段落锚点 / 当前模块:章节号）
  3. module-coverage-matrix.md 对应单元格 ☐ → ◐ 或 ☐ → ☑
  4. blind-bazi-cases-index.md 对应位置同步
```

### §P.3.2 (b) 累计加权 ≥ 2.4 → 进绿灯

```
触发：同一规律来自多个案例 / 多份文档时，候选等级累加超过 2.4
出队动作：
  1. 进入阶段 5 晋级流程（人审 + ≥ 1 个新案例验证）
  2. 通过后走 §P.3.1 流程；不通过则保持在本队列（候选等级冻结）
```

### §P.3.3 (c) 被证伪 → 进 rule-conflicts.md 或降级存疑

```
触发：新案例反馈与该候选规律矛盾，或在多个案例中出现相反兑现
出队动作：
  1. 该条目从本队列移除
  2. rule-conflicts.md 新建争议条目（按争议 #N 编号）
  3. 若被多案验证为伪规律 → 降级存疑（候选等级归零，留作反例标本，参考『丑申合』伪规律修复案例）
```
