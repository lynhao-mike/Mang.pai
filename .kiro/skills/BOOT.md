## 四、加载原则

1. **最小化原则**：启动时只加载路由 + 状态，不预加载任何理论模块
2. **按需原则**：用户意图明确后才加载对应模块，避免无关内容占用上下文
3. **分步原则**：Step 1（速览）仅需 A 模块；Step 3（深度）才追加 B/C/D/E
4. **不重复原则**：同一会话中已加载的模块不重复加载
5. **自检不阻塞**：§0 启动自检在 Boot Set 内完成，不触发额外文件加载

## 五、冷启动流程

```
1. 加载 BOOT.md（本文件）→ 获取加载规则
2. 加载 blind-bazi-analyst.md → 获取路由逻辑 + §0 自检
3. 加载 STATUS.md → 获取持久状态（替代 handoff 中的重复信息）
4. 执行 §0 启动自检 → 静默通过 或 ⚠ 告警
5. 等待用户输入 → 判定模式 → 按需加载对应模块
```
